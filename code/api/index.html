How it works

This poll uses randomized response, a technique for differential privacy. When you submit your answer, a coin is flipped:

With probability p = eε / (1 + eε), your true answer is recorded. Otherwise, the opposite answer is recorded.

This means no one can know for certain what your true answer was — you have plausible deniability. A lower ε means more privacy (more randomness) but noisier aggregate results.

The randomization is done entirely in your browser. Your true answer is never sent to the server — only the (possibly flipped) result is transmitted. You can verify this in your browser's network inspector.

The fixed δ value represents the probability of a complete privacy failure. In pure randomized response it plays no role, but is shown for completeness of the (ε, δ)-DP framing.

A note on privacy

This is not entirely differentially private, because we log the time of your response. In theory, someone with access to the server could know that you were accessing this page at a specific time and correlate that with a recorded response. We won't do that. The 1-hour delay on aggregate results is a partial mitigation, but it is not a complete solution.
